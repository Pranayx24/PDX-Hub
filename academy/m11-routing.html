Phase II: Execution

Routing Phase

The final interconnect. Creating millions of metal paths to connect logic Gates without DRC violations.

1. Global and Detailed Routing

Routing occurs after CTS is finalized. The tool uses a grid-based Manhattan approach (vertical vs horizontal metal layers) to resolve billions of electrical net paths.

Routing Stages:

  • Global Routing (GR): Assigns each net to a series of routing regions (G-Cells) while monitoring congestion. It doesn't create actual metal shapes.
  • Track Assignment: Assigns specific tracks for each net within the G-Cells.
  • Detail Routing (DR): Creates the actual metal geometries and vias. It works on a small window (10x10 tracks) at a time to satisfy all DRC rules.

2. DRC (Design Rule Check) Violations

Detail routing is considered complete only when zero DRC violations remain. Typical routing errors include:

  • Short: Two different nets occupy the same spot on a metal layer.
  • Spacing: Two metals are closer than the foundry's minimum rule (e.g., < 40nm).
  • Antenna Effect: A long metal wire accumulates static charge during manufacturing that can blow the gate oxide of the transistor it's connected to. Fix: Inserting an antenna diode.

Interview focus: Routing Congestion

"What is a G-Cell, and how does the tool use it for routing?"

Answer: G-Cells (Global Cells) are virtual buckets the tool uses to estimate routing density across the chip. By checking the number of nets entering/exiting each G-Cell, the tool can identify congestion hotspots during global routing.
