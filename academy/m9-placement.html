Phase II: Execution

Placement Engine

The positioning of logic. Distributing millions of standard cells within the core to achieve optimal PPA.

1. Global vs Detailed Placement

Placement occurs in two primary stages. The EDA tool uses force-directed algorithms to pull cells toward their logical neighbors while pushing them apart to avoid density hotspots.

Placement Stages:

  • Global Placement (GP): Distributes cells to minimize wirelength; allows cells to overlap temporarily.
  • Detailed Placement (DP/Legalize): Snaps cells to rows and ensures zero overlaps. This is where Tie-Cells are inserted to handle floating pins.
  • Scan Reordering: Optimizing the physical sequence of flip-flops in a scan chain to reduce overall congestion. This effectively "re-wires" the DFT logic to match the physical proximity.

2. Advanced Placement Tactics

Placement isn't just about cells; it's about wires. If nets can't be routed later, the placement is useless.

  • Magnet Placement: Manually pulling specific cells closer to I/O ports or fixed macros using tool attributes to minimize critical path delay.
  • Region-Based Placement: Forcing related logic into a specific core geometry (e.g., create_bounds) to improve performance and local density.
  • Cell Swelling: Artificially increasing the area of a cell to reduce local routing density without using explicit routing blockages.
The Congestion Map

Post-placement, the tool generates a heat map showing "Overflows". If the G-cell overflow is > 1-2%, the design is likely unroutable and needs more cell padding or a better macro arrangement.

Interview focus: Congestion

"What is the difference between Pin Density and Cell Density?"

Answer: Cell Density refers to how much core area is occupied by cells. Pin Density refers to the concentration of connection points (pins). A region with low cell density but many high-pin-count cells (complex gates) can still have huge routing congestion.
