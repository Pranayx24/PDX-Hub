TCL Scripting

The language of EDA. Mastering tool automation to increase PD productivity and eliminate manual effort.

1. Why TCL for EDA?

Every major tool From Synopsys (Design Compiler, IC Compiler II) and Cadence (Genus, Innovus) uses Tool Command Language (TCL) as its primary interface. It allows for complex design exploration that would be impossible via GUI.

High-Performance Collections:

EDA tools handle millions of instances. Using standard TCL lists is too slow. Instead, we use EDA Collections.

# Efficiently iterating over millions of cells:
foreach_in_collection cell [get_cells -hier *] {
    set area [get_attribute $cell area]
    if {$area > 10.0} { echo "Cell is large!" }
}

2. Mastering Attributes & Custom Reporting

Every object in an EDA tool (Net, Pin, Cell) has dozens of Attributes. Understanding how to query and modify these is the "Superpower" of a PD engineer.

  • get_attribute: Extracting real-time data like is_clock_used, slack, or location.
  • set_attribute: Manually overriding the tool's behavior (e.g., set_attribute [get_cells ...] is_fixed true to freeze placement).
  • Custom Scripting: Designing personal timing reports that filter paths based on specific hierarchy or pin-types, helping you find root causes faster than the standard tool report.

Interview focus: Collection Search

"What is the difference between a TCL list and an EDA collection?"

Answer: A TCL list is a standard string-based array. An EDA collection (created by get_cells etc.) is a pointer-based object that handles massive design data efficiently. To iterate over a collection, you must use foreach_in_collection instead of standard foreach in some tools.
