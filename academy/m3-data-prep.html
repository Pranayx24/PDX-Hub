Foundations Phase

Data Preparation

The first step in any Physical Design project. Mastering the inputs that drive the EDA tool flow.

1. The Logical & Physical Inputs

Every PD tool requires two distinct sets of data. Beyond the basic .lib and .lef, we need to define the Power Intent and Parasitic Models.

Logical View & Corners

Libraries are provided for multiple PVT Corners:

  • SS (Slow-Slow): Max Temp, Min Voltage. Used for Setup.
  • FF (Fast-Fast): Min Temp, Max Voltage. Used for Hold.
  • TT (Typical): Average conditions for power estimation.

Physical View (LEF/GDS)

Technology LEF defines the metal stack (M1-M11), routing tracks, and via rules. Cell LEF defines the physical boundary of the logic.

Advanced Power Intent: UPF

The Unified Power Format (UPF) script defines the power architecture of the chip:

  • Power Domains: Grouping logic into different voltage islands.
  • Isolation Cells: Ensuring that an "off" domain doesn't pass floating signals to an "on" domain.
  • Level Shifters: Translating logic levels between domains running at different voltages (e.g., 0.8V to 1.2V).
Parasitic Modeling: SPEF and TLU+

The tool uses RC Tech Files (itf/tlu+) to calculate the resistance and capacitance of wires. The output of extraction is a **Standard Parasitic Exchange Format (SPEF)** file, which is then used by STA tools for timing calculation.

Interview focus: SDC and Timing

"What is the difference between a create_clock and a create_generated_clock?"

Answer: create_clock is used for primary input clock source (PLL output or external pad), while create_generated_clock is used when an internal circuit (like a divider) derives a clock from a master source.

Interview focus: Libraries

"Why do foundries provide multiple Vt variants (LVT, RVT, HVT)?"

Answer: LVT (Low Vt) cells are fast but have high leakage. HVT (High Vt) cells are slow but have nearly zero leakage. By mixing them, we can use LVT only on critical paths and HVT elsewhere to save power.
