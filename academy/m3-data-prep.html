Data Preparation

The first step in any Physical Design project. Mastering the inputs that drive the EDA tool flow.

1. The Logical & Physical Inputs

Every PD tool (Innovus/IC Compiler II) requires two distinct sets of data to define your chip:

Logical View (.lib / .db)

Contains timing, power, and area information for every standard cell. Defines input capacitance, output drive strength, and leakage power.

Physical View (.lef)

Contains cell dimensions (width/height), pin locations, and metal layer obstruction info (antenna properties). Crucial for placement and routing.

Required Files Checklist:
  • Gate Level Netlist (.v): The logic description output from synthesis.
  • Technology File (.tf/.itf): Defines the metal layers, thickness, and resistance properties for the process node (e.g., TSMC 7nm).
  • SDC File (.sdc): Timing constraints defining clock frequency and I/O limits.
  • UPF File (.upf): For multi-voltage designs where some blocks need level shifters or isolation cells.

Interview focus: SDC and Timing

"What is the difference between a create_clock and a create_generated_clock?"

Answer: create_clock is used for primary input clock source (PLL output or external pad), while create_generated_clock is used when an internal circuit (like a divider) derives a clock from a master source.
